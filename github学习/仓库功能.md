### Community standards: 项目的社区完整度

####  **Description（项目描述）**

- **作用**：一句话概括项目功能与目标，放在仓库首页最显眼位置，让访问者快速了解项目价值。

####  **README**

- **作用**：项目的 “说明书”，是新用户接触项目的第一个入口。

- 核心内容

  ：

  - 项目简介、功能亮点
  - 安装步骤、快速上手示例
  - 贡献方式、许可证信息

- **价值**：清晰的 README 能显著降低用户的学习和参与门槛。

####  **Code of conduct（行为准则）**

- **作用**：定义社区成员的行为规范，确保协作环境友好、包容、无骚扰。

- 常见内容

  ：

  - 尊重他人、包容不同背景
  - 禁止歧视、骚扰等不当行为
  - 举报与处理流程

- **价值**：建立健康的社区文化，吸引更多元的贡献者。

#### **Contributing（贡献指南）**

- **作用**：指导外部开发者如何参与项目贡献。

- 核心内容

  ：

  - 提交 Bug 报告的规范
  - 发起 Pull Request 的步骤
  - 代码风格、测试要求

- **价值**：标准化协作流程，让新成员也能顺畅参与。

#### **License（许可证）**

- **作用**：明确代码的使用、修改、分发规则，避免法律风险。

- 常见选择

  ：

  - **MIT**：宽松自由，几乎无限制
  - **Apache 2.0**：包含专利授权条款
  - **GPL**：要求衍生作品也开源

- **价值**：保护作者权益，让使用者明确自身权利。

#### **Security policy（安全政策）**

- **作用**：说明如何报告和处理安全漏洞。

- 核心内容

  ：

  - 漏洞报告的联系方式
  - 响应流程与时间承诺
  - 安全更新公告方式

- **价值**：快速响应安全问题，保护用户和项目安全。

####  **Issue templates（Issue 模板）**

- **作用**：规范 Issue 提交格式，引导用户提供关键信息。

- 常见模板类型

  ：

  - Bug 报告模板（含复现步骤、环境信息）
  - 功能请求模板（含需求描述、应用场景）

- **价值**：提升 Issue 质量，帮助维护者更快定位问题。

#### **Pull request template（Pull Request 模板）**

- **作用**：规范 Pull Request 提交格式，提供清晰的上下文。

- 核心内容

  ：

  - 改动目的、关联的 Issue
  - 测试情况、影响范围
  - 文档更新说明

- **价值**：让代码评审更高效，确保每次提交都有完整信息。

### Traffic ：项目的受欢迎程度

#### 📥 Git clones（Git 克隆统计）

- **Clones in last 14 days**：过去 14 天内，这个仓库被 `git clone` 的总次数。
- **Unique cloners in last 14 days**：过去 14 天内，有多少个不同的 IP 地址执行了克隆操作（去重后的人数）。

![image-20260118052415015](仓库功能.assets/image-20260118052415015.png)

#### 👥 Visitors（访客统计）

- **Total views in last 14 days**：过去 14 天内，仓库页面被浏览的总次数（包括同一人多次访问）。
- **Unique visitors in last 14 days**：过去 14 天内，有多少个不同的用户访问过仓库页面（去重后的人数）。

![image-20260118052502985](仓库功能.assets/image-20260118052502985.png)

#### 🔗 Referring sites（来源网站）

显示用户是通过哪些外部网站链接跳转到你的 GitHub 仓库的，比如博客、社交媒体、技术论坛等。

这可以帮你了解项目的曝光渠道。

![image-20260118052554869](仓库功能.assets/image-20260118052554869.png)

#### 📄 Popular content（热门内容）

显示仓库里哪些文件或页面被访问得最多，比如 README、文档、核心代码文件等。

这能帮你知道用户最关注项目的哪些部分。

![image-20260118052632740](仓库功能.assets/image-20260118052632740.png)

### commits:

- 标题

  ```
  Commits over the last year of yzhzuiliubi/Hello-World
  ```

  

  表示这是过去一年内，该仓库每周的代码提交（Commit）次数统计。

- **时间轴**：横轴从去年 2 月到今年 1 月。

- **提交次数**：纵轴是每周提交的次数，只有在最右侧的 “Jan”（1 月）这一周，有 **2 次提交**，其余所有周的提交次数都是 0。

![image-20260118052826760](仓库功能.assets/image-20260118052826760.png)

### code frequency: 展示该仓库历史上每周的 **代码新增量（Additions）\**和\** 删除量（Deletions）**。

**颜色说明**：

- **绿色区域**：表示当周新增的代码行数（Additions）。
- **红色区域**：表示当周删除或修改的代码行数
